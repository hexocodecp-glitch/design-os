# Design OS Master v1

## 1. Mission

Design OS는 브랜드 데이터를 축적하고 학습하여 디자인, 마케팅, 콘텐츠 제작, 업무 자동화의 성과를 지속적으로 향상시키는 AI 운영 시스템이다.

Design OS의 핵심 목적은 단순히 예쁜 디자인을 만드는 것이 아니라, 브랜드 목표와 고객 행동을 연결하여 실제 성과를 만드는 것이다.

> 디자인은 목적이 아니다. 디자인은 브랜드 목표를 달성하기 위한 수단이다.

---

## 2. Core Philosophy

모든 디자인과 자동화 작업은 다음 기준을 기반으로 한다.

- 브랜드 전략
- 고객 심리
- 제품 USP
- 경쟁사 분석
- 플랫폼 알고리즘
- SEO / GEO 관점
- 고객 언어
- 성과 데이터
- 원본 자산 보호
- 반복 학습

## 4. Brand Archive

Brand Archive는 브랜드의 기억 저장소다.

저장 항목:

- 브랜드명
- 회사명
- 브랜드 포지셔닝
- 브랜드 철학
- 타깃 고객
- 고객 문제
- 고객 욕구
- 브랜드 톤앤매너
- 컬러 시스템
- 타이포그래피
- 금지 표현
- 추천 이미지 스타일
- 경쟁 브랜드
- 주요 상품군
- SNS / 상세페이지 / 스토어 링크

Brand Archive는 완성 문서가 아니라 프로젝트가 진행될수록 계속 업데이트되는 살아있는 문서다.

## 5. Brand Style Pack

Brand Style Pack은 제품 사진이나 실제 촬영 이미지가 없는 상황에서도 브랜드 일관성을 유지하기 위한 스타일 기준 문서다.

사용 상황:

- 정보성 카드뉴스
- 브랜드 스토리 콘텐츠
- 블로그 썸네일
- 광고 배너
- 캠페인 컨셉 이미지
- 제품 사진 없이 만드는 AI 이미지

저장 항목:

- 브랜드 포지션
- 무드 키워드
- 비주얼 키워드
- 컬러 팔레트
- 폰트 방향
- 사진 스타일
- 그래픽 스타일
- 레이아웃 규칙
- AI 이미지 생성 규칙
- 금지 스타일
- 플랫폼별 규칙
- QA 체크리스트

원칙:

> 사진이 없어도 브랜드 스타일은 있어야 한다.

저장 항목:

- 카테고리
- 핵심 USP
- 주요 기능
- 고객 문제
- 고객 사용 상황
- 상세 스펙
- 인증서
- 구성품
- 원본 제품 이미지
- 상세페이지 이미지
- FAQ
- 경쟁 제품 비교
- 주의해야 할 표현

## 7. Product Master Pack

Product Master Pack은 특정 제품을 반복 제작 가능한 자산으로 고정하는 문서다.

사용 상황:

- 카드뉴스 반복 제작
- 상세페이지 제작
- 대표이미지 제작
- 광고 배너 제작
- AI 이미지 생성
- Figma 템플릿 제작

저장 항목:

- 제품명
- 핵심 포지셔닝
- USP 우선순위
- 확정 스펙
- 고객 문제
- 고객 언어 규칙
- 카드뉴스 구조
- 디자인 시스템
- 제품 보존 규칙
- 이미지 생성 규칙
- QA 체크리스트

예시:

```text
products/lantons/auto_shaker/product_master_pack_v1.md
```

## 12. Winning Library

- 높은 CTR
- 높은 CVR
- 높은 저장률
- 높은 공유율
- 높은 댓글 반응
- 높은 체류시간
- 매출 증가

저장 항목:

- 성공한 후킹
- 성공한 카드뉴스 구조
- 성공한 상세페이지 섹션
- 성공한 CTA
- 성공한 이미지 스타일
- 성공한 가격 표현

## 14. Learning System

모든 프로젝트 종료 후 학습 기록을 남긴다.

프로젝트 종료 후 질문:

1. 무엇이 잘 작동했는가?
2. 무엇이 작동하지 않았는가?
3. 고객은 어떤 언어에 반응했는가?
4. 다음 프로젝트에 반영할 점은 무엇인가?
5. Brand Archive에 업데이트할 내용은 무엇인가?
6. Winning Library 또는 Failure Library에 저장할 내용은 무엇인가?

## 15. Asset Lock Rule

업로드된 원본 자산은 변경하지 않는다.

원본 자산 예시:

- 제품 이미지
- 로고
- 패키지
- 인증서
- 상세페이지 캡처
- 인물 사진
- 캐릭터

허용:

- 배경 변경
- 조명 추가
- 그림자 추가
- 그래픽 요소 추가
- 타이포그래피 배치
- CTA 배치

금지:

- 제품 색상 변경
- 제품 형태 변경
- 로고 변경
- 비율 변경
- 텍스트 왜곡
- 인증 내용 변경
- 존재하지 않는 기능 추가

원칙:

> AI는 원본 자산을 재창조하지 않고, 원본 자산을 보호하며 주변 디자인을 구성한다.

## 16. Asset Upload & Selection Rule

이미지는 처음부터 완벽하게 정리해서 올리지 않아도 된다.

단, 정확도를 높이기 위해 최소한 브랜드별 / 제품별 폴더는 분리한다.

예시:

```text
assets/
├─ lantons/
│  ├─ auto_shaker/
│  │  ├─ raw_uploads/
│  │  ├─ product_photos/
│  │  ├─ lifestyle_photos/
│  │  ├─ detail_page_screenshots/
│  │  └─ selected_for_cardnews/
│  └─ hair_brush/
├─ goodcare/
└─ kinosoop/
```

AI 분류 기준:

- Hero image
- Front product image
- 45-degree product image
- Color lineup
- Mixing image
- Cleaning image
- Lifestyle image
- Office scene
- Gym scene
- Car cup holder scene
- Bag storage scene
- Detail page screenshot
- Review capture
- Competitor reference

원칙:

> 자료는 완벽하게 정리해서 시작하지 않아도 된다. 단, 제품별 폴더만 구분하면 AI가 이후 분류하고 선택할 수 있다.

## 17. Asset Mode System

Design OS는 이미지 제작 방식을 두 가지 모드로 구분한다.

### Mode A. Product Asset Mode

실제 제품이 존재하고, 제품 이미지가 필요한 경우.

예시:

- LANTONS Auto Shaker
- LANTONS Hair Brush
- GOODCARE Massage Gun

규칙:

- 제품 원본 필수
- 제품 형태 변경 금지
- 로고 변경 금지
- 색상 변경 금지
- 비율 변경 금지
- 제품을 새로 생성하지 않기

### Mode B. Concept Asset Mode

제품 사진 없이 컨셉, 정보, 감성, 상황 이미지만으로 콘텐츠를 만드는 경우.

예시:

- 키노숲 육아 정보 카드뉴스
- 브랜드 스토리 콘텐츠
- 블로그 썸네일
- 마케팅 정보 콘텐츠
- 캠페인 컨셉 비주얼

규칙:

- 제품 원본 불필요
- Brand Style Pack 필수
- 검증되지 않은 시설, 기능, 인증, 제품 스펙 생성 금지
- 브랜드 컬러, 무드, 조명, 레이아웃 일관성 유지

## 18. QA Engine

출력 전 반드시 검수한다.

검수 항목:

- 제품 이미지가 원본과 같은가?
- 로고가 변경되지 않았는가?
- 색상과 비율이 유지되었는가?
- 수치와 스펙이 사실과 일치하는가?
- 브랜드명과 제품명이 정확한가?
- 플랫폼 사이즈에 맞는가?
- 모바일에서 읽히는가?
- CTA가 명확한가?
- 과장 광고 요소가 없는가?
- Concept Asset Mode일 경우 브랜드 스타일팩과 일치하는가?
- 실제 장소/시설을 허위로 묘사하지 않았는가?

## 23. Platform Rules

플랫폼별로 목적이 다르다.

### SmartStore Detail Page

- 전환 중심
- USP 즉시 제시
- 비교표 중요
- 리뷰와 인증 중요
- FAQ 중요
- 제품 원본 정확도 중요

### Instagram Card News

- 저장 가치 중심
- 첫 장 후킹 중요
- 1장 = 1메시지
- 1장 = 1비주얼
- 1장 = 1감정
- 감성 + 정보 균형
- 1080 x 1350px 4:5 권장

### Blog Image

- 검색 유입 보조
- 썸네일 가독성 중요
- SEO 키워드 반영
- Concept Asset Mode 활용 가능

### Ad Banner

- 3초 이해
- 강한 CTA
- 핵심 혜택 단일화
- 제품형은 Product Asset Mode 우선
