# Figma Design Approval Workflow v1

이 문서는 Design OS 및 업무자동화 시스템에서 Figma 작업을 진행할 때 반드시 지켜야 하는 승인 절차를 정의한다.

---

## 1. Core Rule

앞으로 모든 Figma 제작 작업은 반드시 아래 순서를 따른다.

```text
1. 기획서 작성
2. 사용자 검토
3. 승인 또는 수정 요청 반영
4. 최종 기획서 확정
5. Figma 제작
6. Figma 결과 검토
7. 수정 반영
8. GitHub 기록
```

즉, 사용자가 명시적으로 바로 제작을 요청하지 않는 한, Figma 작업은 기획서 없이 바로 진행하지 않는다.

---

## 2. Applies To

이 규칙은 다음 작업에 모두 적용한다.

```text
상세페이지 디자인
홈페이지 디자인
랜딩페이지 디자인
카드뉴스 Figma 레이아웃
브랜드 소개 페이지
판매 페이지
업무자동화 시스템 UI
대시보드 UI
관리자 페이지 UI
모바일 / 태블릿 / PC 반응형 화면
```

---

## 3. Required Planning Document

Figma 제작 전 기획서에는 최소한 다음 항목이 포함되어야 한다.

```text
1. 제작 목적
2. 대상 고객
3. 화면 사이즈
4. 핵심 메시지
5. 메인 카피
6. 서브 카피
7. 섹션 구조
8. 레이아웃 방향
9. 디자인 스타일
10. 컬러 방향
11. 폰트 방향
12. 이미지 사용 방식
13. 피해야 할 표현
14. QA 체크리스트
15. Figma 작업 지시서
```

---

## 4. User Review Rule

기획서 작성 후에는 반드시 사용자에게 검토를 요청한다.

사용자에게 확인해야 할 항목:

```text
카피 방향이 맞는가?
스타일 방향이 맞는가?
구매 전환 또는 목적에 적합한가?
너무 과장되거나 어색한 표현은 없는가?
상품/브랜드 정보가 정확한가?
Figma로 제작해도 되는가?
```

사용자가 승인하기 전에는 Figma 제작을 시작하지 않는다.

---

## 5. Approval Keywords

다음과 같은 표현이 있을 때 Figma 제작 승인으로 본다.

```text
좋습니다. 피그마로 만들어주세요.
승인합니다.
이대로 제작해주세요.
피그마 진행해주세요.
이 기획서로 진행해주세요.
```

다만 사용자가 이전에 “항상 검토 후 제작” 규칙을 설정했으므로, 기획서가 없는 상태에서 “피그마로 만들어주세요”라고 요청하면 먼저 기획서를 작성하고 확인을 받아야 한다.

---

## 6. Exception Rule

사용자가 명확하게 다음처럼 말한 경우에만 기획서 없이 바로 Figma 작업을 진행할 수 있다.

```text
기획서 생략하고 바로 피그마로 만들어주세요.
검토 없이 바로 제작해주세요.
테스트용이라 바로 만들어주세요.
```

이 경우에도 결과물에는 Designer Notes 또는 제작 메모를 남긴다.

---

## 7. Figma Production Rule

Figma 제작 시에는 다음 기준을 따른다.

```text
텍스트는 모두 편집 가능한 레이어로 제작한다.
이미지 영역은 실제 사진 교체가 가능하도록 구성한다.
중요 정보는 모바일에서도 읽히는 크기로 만든다.
제품 정보가 불확실하면 placeholder 또는 확인 필요 문구를 남긴다.
레이어 이름은 이해하기 쉽게 작성한다.
기존 버전은 가능하면 삭제하지 않고 V1, V2 형태로 비교 가능하게 남긴다.
```

---

## 8. Responsive Design Rule

홈페이지와 웹 UI는 기본적으로 다음 3가지 사이즈를 고려한다.

```text
PC: 1440px 기준
Tablet: 768px 기준
Mobile: 390px 기준
```

상세페이지는 플랫폼 기준에 따라 별도 사이즈를 적용한다.

예시:

```text
네이버 스마트스토어 상세페이지: 860~870px width 기준
인스타그램 카드뉴스: 1080x1350px 기준
```

---

## 9. GitHub Archive Rule

Figma 작업이 발생하면 관련 프로젝트 폴더에 다음 내용을 기록한다.

```text
기획서
Figma 파일 URL
버전 변경 내역
수정 요청 사항
최종 승인 여부
QA 결과
```

---

## 10. Current Global Setting

현재부터 Design OS의 Figma 작업 기본 설정은 다음과 같다.

```text
Figma 작업은 항상 기획서 작성 → 사용자 검토 → 승인 후 제작 순서로 진행한다.
```
