# AI디자인 공통규칙

## 1. 목적
AI를 활용한 상세페이지, 카드뉴스, 대표이미지, 홈페이지, 블로그 이미지, Figma 디자인 작업의 공통 기준을 정의한다.

## 2. 적용 범위
- 상세페이지
- 카드뉴스
- 대표이미지
- 홈페이지
- 랜딩페이지
- Figma 디자인
- 블로그 이미지
- 스마트스토어 콘텐츠

## 3. 핵심 원칙
디자인은 목적이 아니라 브랜드 목표를 달성하기 위한 수단이다.

## 4. 기본 작업 순서
1. 목적 확인
2. 자료 확인
3. 기획서 작성
4. 사용자 검토
5. 승인 후 제작
6. QA 검수
7. 수정 반영
8. 기록 저장

## 5. Figma 제작 규칙
Figma 작업은 항상 기획서 작성 → 사용자 검토 → 승인 후 제작 순서로 진행한다.

## 6. 제품 이미지 규칙
제품 사진이 있는 경우 제품 형태, 색상, 로고, 비율을 임의로 변경하지 않는다.

제품 사진이 있는 경우에는 AI가 제품을 새로 만들지 않는다.
실제 제품 사진을 우선 사용하고, 배경·레이아웃·텍스트만 디자인한다.

제품 사진이 들어갈 위치는 필요 시 `사진 슬롯`으로 구성한다.

예시:

```text
MAIN_PRODUCT_IMAGE_SLOT
MAIN_PRODUCT_IMAGE_SLOT_RIGHT
SUB_PRODUCT_IMAGE_SLOT_LEFT_TOP
SUB_PRODUCT_IMAGE_SLOT_LEFT_BOTTOM
SUB_PRODUCT_IMAGE_SLOT_TOP
```

사진 슬롯 규칙:

```text
1. 사진이 들어갈 전용 프레임을 만든다.
2. 슬롯 이름을 명확하게 작성한다.
3. Clip Content ON 구조로 만든다.
4. 사진만 교체해도 레이아웃이 유지되게 한다.
5. 제품 사진과 텍스트 영역을 분리한다.
6. 사진은 왜곡 없이 비율을 유지한다.
```

## 7. 텍스트 규칙
텍스트 규칙은 제작 방식에 따라 `Figma 버전`과 `이미지 생성 버전`으로 구분한다.

---

### 7-1. Figma 버전 텍스트 규칙
Figma로 제작하는 작업에서는 중요한 한글 텍스트를 반드시 편집 가능한 텍스트 레이어로 제작한다.

적용 대상:

```text
상세페이지 메인 카피
상품명
브랜드명
가격
스펙
구매 포인트
카드뉴스 제목
CTA 문구
주의사항
리뷰 문구
인증/보증/배송 안내
```

Figma 텍스트 규칙:

```text
1. 모든 중요한 한글 텍스트는 Figma 텍스트 레이어로 작성한다.
2. 이미지 안에 중요한 텍스트를 합성하지 않는다.
3. 사용자가 수정할 수 있도록 텍스트를 분리한다.
4. 상품명, 브랜드명, 모델명, 숫자, 스펙은 오타 없이 작성한다.
5. 텍스트 레이어 이름은 의미 있게 작성한다.
6. 모바일에서도 읽히는 크기를 유지한다.
7. 최종 제작 전 텍스트 QA를 반드시 진행한다.
```

Figma 작업의 핵심 원칙:

```text
중요한 텍스트는 이미지가 아니라 Figma에서 작성한다.
```

---

### 7-2. 이미지 생성 버전 텍스트 규칙
이미지 생성 작업에서는 한글 텍스트 오류, 오타, 깨진 글자, 이상한 문자가 발생할 수 있으므로 중요한 텍스트를 이미지 생성 모델에 직접 맡기지 않는다.

이미지 생성 시 기본 원칙:

```text
1. 이미지 생성 프롬프트에는 긴 한글 문장을 넣지 않는다.
2. 중요한 카피, 상품명, 가격, 스펙은 이미지 안에 생성하지 않는다.
3. 이미지 생성은 배경, 제품 연출, 분위기, 인물, 소품, 장면 구성 중심으로 사용한다.
4. 텍스트는 후처리 단계에서 Figma로 넣는다.
5. 이미지 안에 텍스트가 필요하면 짧은 단어만 사용한다.
6. 생성된 이미지에 오타나 이상한 글자가 있으면 사용하지 않는다.
7. 텍스트가 깨진 이미지는 반드시 수정 또는 재생성한다.
```

이미지 생성에서 허용 가능한 텍스트:

```text
짧은 영문 로고
간단한 숫자
아이콘 수준의 짧은 라벨
배경에 흐릿하게 들어가는 장식용 텍스트
```

이미지 생성에서 금지할 텍스트:

```text
한글 메인 카피
상품명
브랜드명
가격
모델명
스펙
인증 문구
리뷰 문구
배송/교환/환불 안내
상세페이지 핵심 문장
```

이미지 생성 작업의 핵심 원칙:

```text
이미지 생성은 비주얼을 만들고,
텍스트는 Figma에서 정확하게 넣는다.
```

---

### 7-3. 텍스트 포함 이미지가 꼭 필요한 경우
이미지 자체에 텍스트가 반드시 들어가야 하는 경우에는 아래 순서를 따른다.

```text
1. 먼저 텍스트 없는 이미지 생성
2. Figma에서 정확한 텍스트 삽입
3. 텍스트 오타 검수
4. 모바일 가독성 확인
5. 최종 PNG로 export
```

예외적으로 이미지 생성 단계에서 텍스트를 넣어야 할 경우에는 다음 조건을 따른다.

```text
1. 텍스트는 1~3단어 이내로 제한한다.
2. 한글보다 영문을 우선한다.
3. 생성 후 반드시 오타 검수를 한다.
4. 오타가 있으면 해당 이미지는 최종본으로 사용하지 않는다.
```

---

### 7-4. 텍스트 QA 체크리스트
최종 결과물 제작 전 반드시 아래 항목을 확인한다.

```text
브랜드명이 정확한가?
상품명이 정확한가?
모델명이 정확한가?
숫자와 단위가 정확한가?
한글 오타가 없는가?
깨진 글자나 이상한 문자가 없는가?
문장 줄바꿈이 자연스러운가?
모바일에서 읽을 수 있는가?
이미지 안에 잘못 생성된 텍스트가 없는가?
Figma 텍스트 레이어로 수정 가능한가?
```

최종 원칙:

```text
정확해야 하는 텍스트는 Figma로 작성한다.
이미지 생성은 텍스트 없는 비주얼 제작에 집중한다.
오타가 있는 이미지는 최종 결과물로 사용하지 않는다.
```

## 8. 정보 정확성 규칙
검증되지 않은 스펙, 후기, 인증, 수상 내역, 효능, 성능을 임의로 작성하지 않는다.

제품 정보가 불확실한 경우에는 반드시 아래처럼 표시한다.

```text
확인 필요
실제 상품 기준 확인 필요
재고 상태 기준 확인 필요
옵션 및 상세 안내 기준 확인 필요
```

## 9. 스타일 규칙
브랜드 톤과 제품 목적에 맞는 스타일을 우선한다.

스타일 선택 시 반드시 아래를 확인한다.

```text
상품 특성에 맞는가?
구매 고객에게 적합한가?
브랜드 이미지와 맞는가?
플랫폼에 적합한가?
가독성이 충분한가?
```

## 10. 금지사항
- 제품 왜곡
- 브랜드명 오타
- 상품명 오타
- 모델명 오타
- 허위 스펙
- 과장 표현
- 승인 전 Figma 제작
- 무근거 후기 생성
- 이미지 생성 모델에 중요한 한글 텍스트 직접 생성 요청
- 오타가 있는 이미지 최종 사용
- 실제 제품과 다른 이미지 사용

## 11. QA 체크리스트
- 목적이 명확한가?
- 제품 정보가 정확한가?
- 브랜드명이 정확한가?
- 상품명이 정확한가?
- 모델명이 정확한가?
- 카피가 읽기 쉬운가?
- 한글 오타가 없는가?
- 모바일에서 가독성이 확보되는가?
- 중요한 텍스트가 Figma 텍스트 레이어로 작성되었는가?
- 이미지 안에 깨진 글자나 이상한 문자가 없는가?
- 사용자가 승인한 방향과 일치하는가?
